# ایکس-ری وی‌پی‌ان در سرویس‌های وب آمازون

اسکریپت Terraform برای استقرار سرور وی‌پی‌ان ایکس-ری در AWS با صدور خودکار گواهی TLS.

## ویژگی‌ها
- استقرار خودکار سرور وی‌پی‌ان ایکس-ری در AWS
- صدور خودکار گواهی TLS با استفاده از Let's Encrypt
- ادغام DNS پویا با DuckDNS
- انتقال WebSocket با TLS
- قابل تنظیم برای چندین سرور

## پیش‌نیازها
- نصب Terraform CLI (نسخه 1.3.9 یا بالاتر)
- نصب AWS CLI
- حساب AWS با دسترسی‌های مناسب
- حساب DuckDNS و توکن (به دستورالعمل‌های تنظیم زیر مراجعه کنید)
- جفت کلید SSH تولید شده

### تنظیم اعتبارنامه‌های AWS
1. **ایجاد کلیدهای دسترسی AWS**
   - وارد کنسول AWS خود شوید
   - به IAM → Users → Your User → Security credentials بروید
   - یک کلید دسترسی جدید ایجاد کنید و شناسه کلید دسترسی و کلید مخفی را یادداشت کنید

2. **پیکربندی پروفایل AWS**
   فایل `~/.aws/credentials` خود را با اعتبارنامه‌های AWS خود ایجاد یا ویرایش کنید:
   ```ini
   [xray_profile]
   aws_access_key_id = YOUR_ACCESS_KEY_ID
   aws_secret_access_key = YOUR_SECRET_ACCESS_KEY
   ```
   
   به طور جایگزین، می‌توانید یک نام پروفایل متفاوت در `terraform.tfvars` تنظیم کنید:
   ```bash
   profile = "your-custom-profile-name"
   ```

### تنظیم DuckDNS
1. **ایجاد حساب DuckDNS**
   - به [DuckDNS.org](https://www.duckdns.org) مراجعه کنید
   - با حساب دلخواه خود (Google، GitHub و غیره) وارد شوید

2. **ایجاد یک زیردامنه**
   - در داشبورد DuckDNS، نام زیردامنه مورد نظر خود را وارد کنید
   - روی "add domain" کلیک کنید (مثلاً اگر "myserver" وارد کنید، "myserver.duckdns.org" دریافت خواهید کرد)
   - نام کامل دامنه خود را یادداشت کنید

3. **دریافت توکن خود**
   - توکن DuckDNS شما در بالای صفحه داشبورد نمایش داده می‌شود
   - این توکن را کپی کنید - برای فایل `terraform.tfvars` به آن نیاز دارید

4. **تست تنظیمات خود (اختیاری)**
   می‌توانید تنظیمات DuckDNS خود را با اجرای این دستور تست کنید:
   ```bash
   curl "https://www.duckdns.org/update?domains=YOUR_SUBDOMAIN&token=YOUR_TOKEN&ip=1.2.3.4"
   ```

## شروع سریع

1. **کلون و پیکربندی**
   ```bash
   git clone https://github.com/mmshad/xray-aws-terraform.git
   cd xray-aws-terraform
   cp terraform.tfvars.example terraform.tfvars
   ```

2. **ویرایش پیکربندی**
   فایل `terraform.tfvars` را با مقادیر خود ویرایش کنید:
   ```bash
   domain = "your-subdomain.duckdns.org"  # دامنه DuckDNS شما از تنظیمات بالا
   email  = "your-email@example.com"      # ایمیل برای گواهی‌های Let's Encrypt
   token  = "your-duckdns-token"          # توکن DuckDNS شما از داشبورد
   ```

3. **استقرار**
   ```bash
   terraform init
   terraform validate
   terraform plan
   terraform apply
   ```

4. **دریافت جزئیات اتصال**
   پس از استقرار، لینک اتصال سبز رنگ را در خروجی پیدا کنید:
   - برای پروتکل VMESS: لینک `vmess://`
   - برای پروتکل VLESS: لینک `vless://`
   
   این لینک را به کلاینت V2Ray/X-Ray خود کپی کنید.

## پیکربندی

### متغیرهای ضروری
- `domain`: زیردامنه DuckDNS شما (مثلاً "myserver.duckdns.org") - از داشبورد DuckDNS دریافت شده است
- `email`: ایمیل برای ثبت گواهی Let's Encrypt - برای اعلان‌های تمدید گواهی استفاده می‌شود
- `token`: توکن DuckDNS شما برای به‌روزرسانی DNS - در داشبورد DuckDNS یافت می‌شود

### متغیرهای اختیاری
- `region`: منطقه AWS (پیش‌فرض: "us-east-1")
- `ec2_count`: تعداد سرورهای وی‌پی‌ان (پیش‌فرض: 1)
- `instance_type`: نوع نمونه EC2 (پیش‌فرض: "t2.micro")
  > **توجه**: `t2.micro` ممکن است در همه مناطق AWS در دسترس نباشد. اگر با مشکلات دسترسی مواجه شدید، از `t3.micro` استفاده کنید که عملکرد بهتری دارد اما ممکن است هزینه کمی بیشتر داشته باشد. به عنوان مثال، `t2.micro` در `eu-north-1` در دسترس نیست.
- `protocol`: پروتکل X-Ray برای استفاده - "vmess" یا "vless" (پیش‌فرض: "vmess")
- `profile`: نام پروفایل AWS CLI (پیش‌فرض: "xray_profile")

## چندین سرور
برای استقرار چندین سرور وی‌پی‌ان، مقدار `ec2_count` را در فایل `terraform.tfvars` تنظیم کنید:
```bash
ec2_count = 3
```
هر سرور لینک اتصال مخصوص به خود را در خروجی خواهد داشت.

## انتخاب پروتکل
این پروژه از هر دو پروتکل VMESS و VLESS پشتیبانی می‌کند:

### VMESS (پیش‌فرض)
- پشتیبانی گسترده‌تر توسط کلاینت‌های V2Ray
- استفاده از پروتکل قدیمی با لایه رمزنگاری اضافی
- توصیه شده برای حداکثر سازگاری

### VLESS
- پروتکل جدیدتر و کارآمدتر
- عملکرد بهتر با سربار کمتر
- نیاز به کلاینت‌های جدیدتر V2Ray/Xray

### تغییر پروتکل
برای استفاده از VLESS به جای VMESS، این خط را به فایل `terraform.tfvars` اضافه کنید:
```bash
protocol = "vless"
```

استقرار به طور خودکار لینک اتصال مناسب (`vmess://` یا `vless://`) را بر اساس انتخاب شما تولید می‌کند.

## نکات امنیتی
- تمام ترافیک با TLS رمزگذاری شده است
- پورت‌های تصادفی برای هر استقرار اختصاص داده می‌شوند
- دسترسی SSH برای عیب‌یابی در دسترس است
- گروه‌های امنیتی فقط پورت‌های ضروری را مجاز می‌کنند

## پاکسازی
برای تخریب زیرساخت:
```bash
terraform plan -destroy
terraform destroy
```

- محدودیت‌های خدمات AWS برای نمونه‌های EC2 را بررسی کنید
